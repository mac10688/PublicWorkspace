Problem
In .NET no direct support for File system tree like Windows Explorer. Another problem is to track changes of inserting or removing CDroms, USB drives etc.
Solution
This example show how to build TreeView control for file system's folders.
First of all, create drives folders

Then, every 5 seconds DispatcherTimer is run, which creates new tree using information about which nodes have been already opened. There are 2 functions: TreeView_Update for creating root drives, and TreeView_AddDirectoryItems() for creating folders.
The next important thing is expanding the node:

if (rootItem.Items.Count == 1 && rootItem.Items[0] == null)
{
	rootItem.Items.Clear();

	string[] dirs;
	try
	{
		dirs = Directory.GetDirectories((string)rootItem.Tag);
	}
	catch { return; }

	foreach (var dir in dirs)
	{
		TreeViewItem subItem = new TreeViewItem();
		subItem.Header = new DirectoryInfo(dir).Name;
		subItem.Tag = dir;
		try
		{
			if (Directory.GetDirectories(dir).Length > 0)
				subItem.Items.Add(null);
		}
		catch { }
		subItem.Expanded += TreeViewItem_Expanded;
		rootItem.Items.Add(subItem);
	}
}

 Some folders have no subfolders so we need not to show expander icon for it. But if folder has subfolders, I add as NULL as node element. So expander icon is shown. When node is expanding NULL removing and showing all subfolders.
You can use this example to build more complex file manager. However, it's better to use C++ and native OS support. Because of multiplatform nature of .NET you can't use benefitions of Windows API in .NET.
