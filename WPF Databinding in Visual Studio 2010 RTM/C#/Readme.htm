Note: This sample targets Microsoft .NET Framework 3.5.

This is a Windows Presentation Framework sample that uses an Entity Data Model to connect to a simple Order Management System in a SQL Server database. The sample contains several WPF forms, each demonstrating various data binding features. The sample includes the following:

·         A sample SQL Server database: OMS. A SQL script is included in the sample that can be used to create and populate the database.

·         A data access layer. This class library project has an Entity Data Model (EDM) that maps to the various tables in the database. This also includes some validation logic.

·         The WPF application project. This consumes the data access layer and displays the data. The project contains multiple forms intended to demonstrate various aspects of WPF forms over data, such as:

o    A simple list of items

o    DataGrid

o    Details view with navigation buttons

o    Master-Details

o    Lookup tables

o    Formatting and validation

 

To run this sample

1.     Open the sample project.

2.     Under the Solution node, expand the Solution Items folder and open CreateDatabaseOMS.sql.

3.     Right-click in the TSQL editor and, from the context menu, select Execute SQL.

4.     When prompted to select a connection, choose or create a new connection to your SQL Server.

5.     Close the TSQL Editor

6.     Press F5 to build and run the sample

Requirements

·         Microsoft SQL Server 2005 or above
