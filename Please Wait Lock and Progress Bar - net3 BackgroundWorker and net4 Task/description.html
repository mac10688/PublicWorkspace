Introduction

This example shows the preferred method for locking the screen, when doing time intensive processing.
The old method would be to pop up a window, with a progress bar.
For modern applications it is cleaner and nicer to place a layer on top of the workspace which obscures the workspace and displays the please wait message and/or progress bar.

 

Building the Sample

Just download, unblock, unzip, open and run!

 

Description

This example includes two methods to do this.

 

BackgroundWorker

Firstly, the old .net3.5 BackgroundWorker method.

Because I am using binding, to public properties, which notify binding updates with PropertyChanged events, I don't even have to use a Dispatcher to get back onto the UI thread from the mouse click.

If I didn't use binding, and tried to set the Visibility property directly, it would be a thread exception unless I used a Dispatcher (like Application.Current.Dispatcher) to get back on the UI thread first.

        private void Button_Click(object sender, RoutedEventArgs e)
        {
            Section1Locked = true;

            var bgw = new BackgroundWorker { WorkerReportsProgress = true };
            bgw.RunWorkerCompleted += new RunWorkerCompletedEventHandler(bgw_RunWorkerCompleted);
            bgw.DoWork += new DoWorkEventHandler(bgw_DoWork);
            bgw.ProgressChanged += new ProgressChangedEventHandler(bgw_ProgressChanged);
            bgw.RunWorkerAsync();
        }

Task

It is simpler for .net4's Task class:
C#
Edit|Remove
csharp 
        private void Button_Click_1(object sender, RoutedEventArgs e)
        {
            Section2Locked = true;
            Task.Factory.StartNew(() => 
            {
                for(var a=0; a < 10; a++)
                {
                    Thread.Sleep(500);
                    Progress2 = a * 10;
                }
            })
            .ContinueWith(ret =>
                {
                    Section2Locked = false;
                });
        }
        private void Button_Click_1(object sender, RoutedEventArgs e) 
        { 
            Section2Locked = true; 
            Task.Factory.StartNew(() =>  
            { 
                for(var a=0; a < 10; a++) 
                { 
                    Thread.Sleep(500); 
                    Progress2 = a * 10; 
                } 
            }) 
            .ContinueWith(ret => 
                { 
                    Section2Locked = false; 
                }); 
        }
